基础知识-面向对象

Java程序运行原理

  • java源代码文件–>编译成.class文件(字节码) —>JVM加载.class文件
  • 编译过程: 编译完成的文件放在target文件目录中，javac命令可以实现编译
  • 启动过程: 当你点击开发软件的启动按钮时，IDEA自动帮你拼装了一个命令
  • java的平台无关性是指Java源代码编译后都是字节码，JVM只运行字节码；而只需要在不同操作系统上次构建一个JVM虚拟机，就能实现代码的跨平台

JDK/JRE有什么区别?

  • JDK（Java开发工具包，主要执行Java源代码到字节码的编译）=JRE（Java运行环境，能够将字节码编译成不同系统的底层调用） + javac
  • JDK和JRE都只是一个软件，JVM是一个抽象的概念

Java的基本数据类型

  • byte–>1字节
  • char–>2字节
  • short–>2字节l
  • ong–>8字节
  • int–>4字节
  • float–>4字节
  • double–>8字节
  • boolean–>true/false

Java的参数传递是传值还是传引用

  • Java世界中的一切对象都是指针（地址）
  • 函数调用永远是传值

StringBuffer/StringBuilder的区别/线程安全性？

  • 如果没有额外声明，所有的类默认都是线程不安全的
  • StringBuilder更快，但是线程不安全，更加常用
  • StringBuffer稍慢，但是线程安全(内部的方法都使用了synchronized进行同步)

Object中有那些方法?

其中比较常见的是:

  • equals()
  • hashCode()
  • toString()
  • wait()
  • notify()

String中常见的有那些方法?

  • compareTo(String): int ->比较字符串的大小
  • contains(CharSequence): boolean ->包含某个字符
  • hashCode（）: int ->计算hash值
  • indexOf(String): int ->返回某个字符串的索引位置
  • isEmpty(): boolean ->判断是否是空字符串
  • valueOf(): String ->将其他类型数据转换成String类型
  • split(String): String [ ]->通过正则表达式将字符串进行分割
  • substring(int): String ->截取一定长度的字符串
  • replace(char,char): String -> 替换字符串中的字符
  • ….等

==与equals的区别?

  • ==是比较2个对象的地址，是否是同一个，来判断是否是同一个对象；对Java原生数据，则是比较值是否相等
  • equals则是比较2个对象中的值是否是相等，不是比较是否为同一个对象

深、浅拷贝的区别

浅拷贝

  • 只是将Home这个对象进行了一个拷贝，而其内部的Cat地址仍旧是原来的那个对象；所以相当于只多了一个Home对象而已，不是整个数据链复制一份。

深拷贝

  • 就是将Home这个对象的整个引用链都进行整个拷贝，创建新的对象；所以在浅拷贝的基础上，还需要创建新的Cat对象并赋值给它，还要创建新的字符串“喵喵”让Cat对象指向它，最后让新的Home对象的地址指向新的Cat；就是整个引用链的完全复制。

接口和抽象类有什么区别/联系？

  • Interface：定义功能，只能包含方法（实现），不能包含成员变量；可以被实现若干次;可以理解接口是对功能的抽象，在Java8以后接口可以有默认实现方法
  • Abstract class：定义抽象的骨架实现，可以包含抽象方法（实现），也可以包含成员变量；只能沿着一条路径继承
  • 什么时候用接口/抽象类？
    • 当你所创建的类有成员变量时，只能使用抽象类
    • 如果没有成员变量，接口和抽象类都能实现；只不过接口更为灵活，因为可以多实现，而抽象类就会把你限制在这个继承体系之中了

final语义

  • final在类名上，表示这个类不能再被继承了
  • final在方法名上，表示这个方法不能被覆盖
  • final作用在变量名是，表示这个变量值不能再被更改了

override和overload的区别？

  • overload是重载，指在一个类中是可以存在相同方法名但是引元参数类型不同的方法，当外界调用该方法时，就可以根据传入的参数类型，进行动态的匹配执行那个方法
  • override
    • 重写是子类对父类的允许访问的方法的实现过程进行重新编写,发生在子类中，方法名、参数列表必须相同，返回值范围小于等于父类，抛出的异常范围小于等于父类，访问修饰符范围大于等于父类。
    • 另外，如果父类方法访问修饰符为 private 则子类就不能重写该方法。也就是说方法提供的行为改变，而方法的外貌并没有改变。

Java 面向对象编程三大特性: 封装 继承 多态

  • 封装

    • 封装把一个对象的属性私有化，同时提供一些可以被外界访问的属性的方法，如果属性不想被外界访问，我们大可不必提供方法给外界访问。但是如果一个类没有提供给外界访问的方法，那么这个类也没有什么意义了。

  • 继承

    • 继承是使用已存在的类的定义作为基础建立新类的技术，新类的定义可以增加新的数据或新的功能，也可以用父类的功能，但不能选择性地继承父类。通过使用继承我们能够非常方便地复用以前的代码。
    • 关于继承如下 3 点请记住：
      1. 子类拥有父类对象所有的属性和方法（包括私有属性和私有方法），但是父类中的私有属性和方法子类是无法访问，只是拥有。—> 因为继承实际上是将父类整个类包裹在内部的，只是子类没有权限访问。
      2. 子类可以拥有自己属性和方法，即子类可以对父类进行扩展。
      3. 子类可以用自己的方式实现父类的方法。（以后介绍）。

  • 多态

    • 所谓多态就是指程序中定义的引用变量所指向的具体类型和通过该引用变量发出的方法调用在编程时并不确定，而是在程序运行期间才确定，即一个引用变量到底会指向哪个类的实例对象，该引用变量发出的方法调用到底是哪个类中实现的方法，必须在由程序运行期间才能决定。
    • 简单概括就是，当前程序中对象的引用不确定(有可能是其子类也有可能是它自身的类)，只有当程序运行时，根据具体传入的引用对象去实现调用的方法

    在 Java 中有两种形式可以实现多态：继承（多个子类对同一方法的重写）和接口（实现接口并覆盖接口中同一方法）。

集合框架

前面有篇博客具体讲解了集合体系，所以这里简单提一下。详解集合体系

HashMap原理

  • HashMap基于hash表的存储集合，根据键值对存放数据；初始大小是16，最大容量时2^30，默认加载因子0.75，树化阈值为8（即当一个hash桶内数据对象个数超过8个，转化为红黑树：平衡二叉树），反树化阈值为6个；HashMap是线程不安全的
  • 具体存储是计算键的Hash值，然后存在相应的hash桶中，hash值计算约定中允许不同的对象有相同的hash值，所以一个桶内可能有多个数据对象，默认是链表形式存储；
  • HashMap扩容问题：HashMap当容量超过当前容量与加载因子的乘积后，就会进行自动扩容；扩容为现在容量的2倍，即旧容量左移一位；实际是创建一个新的HaspMap然后将原来的数据进行拷贝过去，这个过程会涉及到进行重新计算hash值、

List/Set/Map区别

  • List：是一个有序的集合
  • Set：是一个无序的集合，并且不允许重复的值存在
  • Map：是以键值对形式存储数据，映射关系，线程不安全

HashMap和HashTable的区别

  • 都是基于哈希表的
  • HashTable是线程安全的（内部方法都使用了synchronized同步），HashMap不是线程安全的
  • HashTable不允许键值为null的存在，HashMap中允许一个键值为null的存在
  • ConcurrentHashMap原理？
    • 在其内部方法过程中使用了synchronized进行同步锁

ArrayList和LinkedList的区别

  • ArrayList底层是由数组实现的，所以在随机访问速度非常快，但是在插入/删除操作慢(除非删除数组尾部)
  • LinkedList底层是由双向链表实现的，插入/删除操作速度非常快，而随机访问速度非常慢(因为存储空间不像数组为一整块地址，所以在遍历其他地址比较耗时)
  • ArrayList 初始大小容量为10，增长大小为原来的容量的一半，及旧的容量加上旧的容量右移一位的大小；最大容量值为Integer最大值-8；

ArrayList/Vector的区别

  • 都是基于数组的自动扩容实现
  • Vector线程安全，但是慢，被废弃了；ArrayList线程不安全，但是效率高

异常

Java的异常体系结构

  • Throwable 任何异常/错误的主线类，checked
    • Exception异常，可以从异常状态中恢复
      • RuntimeException预料之外的异常，表示一个Bug，unchecked
      • 其他Exception预料之中的异常，checked
    • Error 无法恢复的异常状态 unchecked

try/catch/finally的执行顺序？

  • try首先去执行代码块中的程序
  • catch：当try中执行程序时，出现一些异常，将它抓住并做相应处理
  • finally：最后保证都会做的程序，无论前面try和catch的执行；在finally里中的return会覆盖掉try中的return

throw/throws的区别？

  • throw是指在代码中对出一个异常
  • throws是在方法签名上使用，表示这个方法可能会丢出这个checked的异常

final/finally/finalize的区别？

  • final：在类名上，表示类无法被继承；在方法名上，表示无法被覆盖；在变量上，变量的值无法改变
  • finally：在try/catch中使用，保证一些关键操作一定会执行
  • finalize：Object类中的方法，主要是能够手动覆盖这个方法，以便逃脱垃圾回收器的回收；一个对象的生命周期由垃圾管理器自动进行管理的，当一个类退出消亡时，垃圾管理器会调用其finalize（）的方法，进行回收内存空间。

计算机体系原理

进程和线程的区别与联系？

  • 进程：每个进程都有独立的代码和数据空间（进程上下文），进程间的切换会有较大的开销，一个进程包含1–n个线程。（进程是资源分配的最小单位）
    • 进程占用独享的隔离的内存，文件等资源
    • 同一个进程内的线程共享内存/文件等资源
  • 线程：同一类线程共享代码和数据空间，每个线程有独立的运行栈和程序计数器(PC)，线程切换开销小。（线程是cpu调度的最小单位）

Java中的线程和操作系统线程是什么关系？

  • JVM的线程和OS的线程是一一对应的
  • OS负责调度线程

计算机网络

TCP/UDP的区别

  • 基于连接与无连接
  • TCP要求系统资源较多，UDP较少；
  • UDP程序结构较简单 流模式（TCP）与数据报模式(UDP);
  • TCP保证数据正确性，UDP可能丢包
  • TCP保证数据顺序，UDP不保证

TCP的三次握手与四次挥手

  • 三次握手

  • 四次挥手

  • 为什么要三次握手呢?

    • 因为只有三次这样客户端与服务器的应答，才能保证极大概率的连接成功；倘若只有2次，服务器端就无法确认客户端是否收到消息，所以每一方都要先应答一次对方的回复，才能极大概率保证连接成功

数据结构

集合中常见的数据结构

  • ArrayList: 自动扩容的数组，随机查找时间复杂度是常数
  • LinkedList: 双链表，也可以做队列和栈，插入与删除操作时间复杂度是常数
  • HashMap/HashSet: 哈希表
  • TreeMap/TreeSet: 红黑树(平衡二叉树)
  • ConcurrentHashMap: 分段锁+哈希表
  • LinkedHashMap: 链表+哈希表

链表与二叉树查找树的时间复杂度区别

  • ArrayList：寻址O(1)/更新O(n)/二叉查找O(lgn)
  • LinkedList: 寻址O(n)/更新O(1)/查找O(n)
  • 哈希表: 都是O(1)
  • 红黑树: 都是O(lgn)
  • 二叉查找树: O(lgn)

队列和栈分别是什么?

  • 队列特点: FIFO,先进先出；最常见的使用场景，对二叉树进行层次遍历时，将根节点放入队列中，并将其左右孩子也依次放入队列；如此循环
  • 栈特点: FILO,先进后出；最常见的使用场景，方法栈的使用
